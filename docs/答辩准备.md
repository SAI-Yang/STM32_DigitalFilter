# 数字滤波器课设 — 答辩准备

> STM32F103C8T6 数字滤波器：FIR 低通 + Goertzel 频点分析 + FFT 频谱显示

## 〇、基础概念（零基础篇）

### B1：什么是数字信号？什么是模拟信号？

- **模拟信号**：连续变化的物理量，比如声音（空气振动）、温度（水银柱高度）、电压（连续变化）
  - 举例：人说话的声音、收音机天线收到的电磁波
  - 特点：**连续**，任意时刻都有值

- **数字信号**：用 0 和 1 表示的离散数值序列
  - 举例：MP3 文件、手机里的照片、电脑里的视频
  - 特点：**离散**，只在特定时刻有值，每个值用有限个二进制位表示

**本项目的信号流**：信号源（模拟信号）→ ADC 转换为数字信号 → STM32 做数字滤波处理 → 串口/OLED 显示结果

### B2：模拟滤波器 vs 数字滤波器，有什么区别？

| | 模拟滤波器 | 数字滤波器 |
|---|---|---|
| 实现方式 | 电阻、电容、电感、运放 | 代码计算（乘加法） |
| 修改参数 | 换元件、重新焊电路 | 改代码里的系数 |
| 精度 | 受元件误差影响 | 只受计算精度影响 |
| 温度稳定性 | 差（电阻值随温度变化） | 好（芯片不烫就没问题） |
| 体积 | 大（尤其是低频） | 小（写几行代码就行） |
| 成本 | 要买元件 | 只要有个单片机 |

**本项目用的是数字滤波器**：用 STM32 单片机运行代码来实现滤波。老师和硬件工程师路子的同学可能会聊模拟滤波器，你提上面这些区别就行。

### B3：什么是低通滤波器？高通？带通？带阻？

把频率想象成筛子：

- **低通滤波器**：让低频"通过"，挡住高频
  - 例子：低音炮音箱——只让低音喇叭发声
  - **本项目做的就是低通**：500Hz 通过，3000Hz 滤除

- **高通滤波器**：让高频通过，挡住低频
  - 例子：高音喇叭——只让高音通过

- **带通滤波器**：只让某一频段通过
  - 例子：收音机选台——只让某个频率的电台通过

- **带阻滤波器**：挡住某一频段，其余通过
  - 例子：陷波器——去掉 50Hz 工频干扰

直观理解：
```
原始信号 = 500Hz(低频) + 3000Hz(高频)

经过我们的低通滤波器后：
输出信号 ≈ 500Hz(低频保留) + 3000Hz×0.008(几乎被滤掉)
```

### B4：什么是时域？什么是频域？

- **时域**：横轴是时间，纵轴是信号幅值——看到的是"信号怎么随时间变化"
  - 示波器看到的就是时域波形
  - 你的代码里 `buf_a[i]` 按时间顺序存采样值，就是时域

- **频域**：横轴是频率，纵轴是信号幅值——看到的是"信号包含哪些频率成分"
  - 频谱分析仪看到的就是频域
  - 你的 OLED 频谱柱就是频域——每根柱子代表一个频率有多强

**类比理解**：
- 时域 = 一份菜肴的**配料表**（面粉多少、糖多少、盐多少混在一起）
- 频域 = 把混合的配料**分离出来**单独看（哦，这里面有 500Hz 的成分，还有 3000Hz 的成分）

**两者关系**：同一个信号可以从时域看（波形）也可以从频域看（频谱），只是观察角度不同。FFT（快速傅里叶变换）就是把信号从时域**变换**到频域的工具。

### B5：什么是傅里叶变换？FFT 是什么？

**傅里叶变换（FT）**：数学上的一个变换，能把信号从时域变到频域。核心思想是：**任何信号都可以分解为不同频率正弦波的叠加**。

```
方波 = sin(ωt) + 1/3·sin(3ωt) + 1/5·sin(5ωt) + ... 
```

**快速傅里叶变换（FFT）**：傅里叶变换的一种快速算法实现，不是新的数学概念。

本项目用的 Cooley-Tukey 基-2 FFT：
- 输入 512 个时域点 → 输出 256 个频域点（单边谱）
- 利用"分治"思想：把大 DFT 拆成小 DFT，递归计算
- 计算量从 N² 降到 N·log₂N（512²=262K → 512×9=4608，快了 57 倍）

### B6：什么是 Nyquist 采样定理？

这是数字信号处理**最重要**的概念！

**定理**：要无失真地从数字信号恢复原始模拟信号，采样频率必须 ≥ 信号最高频率的 **2 倍**。

```
Fs ≥ 2·Fmax
```

**本项目的参数**：
- 信号最高频率：5000Hz（我们关心的范围）
- 实际采样率：**10000Hz**（= 2 × 5000Hz，刚刚满足）
- 频率分辨率：10000 / 512 ≈ 19.53Hz

**如果不满足会怎样？** 产生**混叠**——高频信号折返到低频段，无法区分。比如用 6000Hz 采样 4000Hz 的信号，采样结果看起来像是 2000Hz 的信号（4000 = 6000 - 2000）。

### B7：ADC 是什么？DAC 又是什么？

**ADC（模数转换器）**：Analog-to-Digital Converter，把模拟电压变成数字数值。
- 本项目用的 STM32F103 内置 ADC：12bit 精度
- 输入 0~3.3V → 输出 0~4095 的整数
- 代码里：`buf_a[i] = adc_dma[i] * 3.3f / 4096.0f` 就是把 ADC 原始值换算成电压

```
0V     → ADC 输出 0
1.65V  → ADC 输出 2048
3.3V   → ADC 输出 4095
```

**DAC（数模转换器）**：反过来，把数字数值变成模拟电压。本项目没用 DAC。

### B8：什么是单片机？STM32F103C8T6 是什么水平？

**单片机**：把 CPU、内存、各种外设（ADC、定时器、串口等）集成在一个芯片上的微型计算机。

**STM32F103C8T6 参数**：
- CPU：ARM Cortex-M3，72MHz
- RAM：20KB（约能存 5000 个 float 数）
- Flash：64KB（存程序代码）
- 价格：~5 元/片
- **比你家路由器芯片弱几十倍，比手机芯片弱几千倍**

所以在这个项目里你要**精打细算每 1KB 内存**——这就是为什么选 512 点而不是 1024 点，这就是为什么手动优化代码。

### B9：什么是 DMA？为什么要用它？

**DMA（直接存储器访问）**：一个"搬运工"硬件模块，可以在**不经过 CPU** 的情况下，把数据从一个地方搬到另一个地方。

**没用 DMA 的情况**：
```
每次 ADC 转换完 → CPU 中断 → 读数据 → 存到数组 → 回去继续等
CPU 一直在等 ADC，啥也干不了
```

**用了 DMA 的情况**：
```
ADC 转换完 → DMA 自动搬运到数组 → 搬完 512 次才通知 CPU
搬运期间 CPU 在干别的（待机、省电）
```

本项目 DMA 配置：
- Circular（循环）模式：运完 512 次自动重新开始
- 配合 TIM3 定时触发 ADC，实现**全自动后台采集**
- CPU 亲的处理信号计算，DMA 负责采集，互不干扰

### B10：什么是定时器？TIM3 在这里干什么？

**定时器**：单片机的"闹钟"，可以精确产生固定间隔的时间信号。

**TIM3 在本项目中的角色**：每隔 100μs 产生一个触发信号，告诉 ADC"可以开始采了"。

```
TIM3 配置：
  PSC=71     → 72MHz 时钟分频为 1MHz（1μs 一个计数）
  ARR=99     → 计数 0→99 后归零，即 100μs 一次
  TRGO=Update → 每次归零时输出触发信号给 ADC
  
  100μs = 10000Hz（这就是采样率 10kHz 的来源）
```

### B11：什么是中断？本项目中用到中断了吗？

**中断**：CPU 正在干活，突然有急事需要处理——CPU 暂停当前工作，先去处理急事，处理完回来继续干活。

项目中用到的中断：
- **原始 ADC 版本**：DMA 传输完成中断 → `HAL_ADC_ConvCpltCallback` → 告诉主循环"数据准备好了"
- **SysTick 中断**：HAL 库的 1ms 时钟节拍，用于 `HAL_Delay(50)` 延时
- 没有使用其他中断（按键中断等本项目没用到）

**为什么软件 I2C 要关中断？** 软件 I2C 的时序是 GPIO 电平精确翻转的，如果中间被中断打断，时序就乱了。所以 OLED_Update 期间尽量不让中断干扰。

### B12：什么是 GPIO？什么是 I2C？

**GPIO**：通用输入输出引脚，单片机最基础的功能——可以把引脚设为高电平（3.3V）或低电平（0V），或者读取引脚上的电平。

**I2C**：一种通信协议，用两根线（SCL 时钟线、SDA 数据线）连接多个设备。本项目用 I2C 连接 OLED 屏幕。

**本项目 OLED 的软件 I2C 实现**：
```c
void i2c_delay(void) { for (volatile int i = 0; i < 8; i++); }
// ↑ 用空循环制造延时，控制 I2C 通信速度

SDA_H / SDA_L：把数据线拉高/拉低（写 GPIO）
SCL_H / SCL_L：把时钟线拉高/拉低（写 GPIO）

SDA_R：读取数据线电平（读 GPIO）
```

软件 I2C 就是用 GPIO 引脚模拟 I2C 协议的时序，完全是"手动挡"操作。优点是移植性好、没有硬件 bug；缺点是 CPU 占用高。

### B13：什么是采样率？什么是频率分辨率？

| 概念 | 定义 | 本项目值 |
|------|------|---------|
| **采样率** Fs | 每秒采集多少个样本点 | 10000 Hz（每秒 10000 次） |
| **采样点数** N | 每帧处理多少个样本 | 512 点 |
| **帧时间** | 采集一帧需要多久 | 512/10000 = 51.2ms |
| **频率分辨率** | 频谱中相邻两根柱子间隔多少 Hz | 10000/512 = 19.53Hz |
| **Nyquist 频率** | 能测量的最高频率 | 10000/2 = 5000Hz |

频率分辨率 19.53Hz 的意思：OLED 上第 1 根柱子代表 0~19.53Hz，第 2 根代表 19.53~39.06Hz……第 26 根代表 500Hz 附近的能量。

### B14：什么是信号的幅值？什么是 dB（分贝）？

**幅值**：信号的大小，可以理解为"音量"。

```
sin(500t)  → 幅值 1.0
0.5·sin(3000t) → 幅值 0.5
```

**分贝（dB）**：用对数表示两个量的比值，用来描述"衰减了多少"。

```
衰减(dB) = 20 × log₁₀(输出幅值 / 输入幅值)

举例：
输出 = 输入        → 0dB（没衰减）
输出 = 输入×0.5   → -6dB（衰减一半）
输出 = 输入×0.1   → -20dB
输出 = 输入×0.01  → -40dB
输出 = 输入×0.001 → -60dB

本项目实测 3000Hz 衰减约 -35dB ≈ 输出是输入的 1/56
```

### B15：我需要在答辩时说清楚哪条"数据流"？

这是整份答辩最核心的主线，一定要能流畅说出来：

```
信号源产生模拟信号
    ↓
隔直电容 + 偏置电路（把信号调理到 ADC 可接受的 0~3.3V 范围）
    ↓
ADC 以 10kHz 采样（或软件直接生成数字信号）
    ↓
DMA 自动搬运 512 个采样值到内存（不占 CPU）
    ↓
FIR 滤波器做低通滤波（保留 500Hz，滤除 3000Hz）
    ↓
Goertzel 算法精确计算 500Hz 和 3000Hz 的幅值（串口输出）
    ↓
FFT 计算全频谱（用于 OLED 频谱显示）
    ↓
OLED 显示频谱柱状图 + 串口输出数值
```

## 一、滤波器基础（必问）

### Q1：FIR 和 IIR 滤波器的区别？为什么选 FIR？

| | FIR | IIR |
|---|---|---|
| 冲激响应 | 有限长 | 无限长 |
| 相位 | **严格线性相位**（所有频率延迟相同） | 非线性相位 |
| 稳定性 | **绝对稳定**（只有零点） | 可能不稳定（有极点） |
| 计算量 | 阶数高，计算量大 | 阶数低，计算量小 |
| 设计方法 | 窗函数法、频率采样法 | 双线性变换、冲激响应不变法 |

**选 FIR 的原因**：数字信号处理课程的核心知识点是 FIR 设计，而且 FIR 具有严格的线性相位特性——信号通过滤波器后各频率分量的相对位置不变，波形不失真。对于课设来说，FIR 的稳定性和线性相位是相比 IIR 最大的优势。

### Q2：汉明窗（Hamming Window）的原理？为什么用 Hamming 不用其他的？

窗函数法设计 FIR：理想低通滤波器的冲激响应是无限长的 sinc 函数，直接截断会产生 Gibbs 现象（通带和阻带有剧烈振荡）。加窗就是用一个窗函数去乘 sinc 函数，平滑截断。

常见窗函数对比：

| 窗函数 | 主瓣宽度 | 最小阻带衰减 | 过渡带 |
|--------|---------|-------------|--------|
| 矩形窗 | 最窄 | -21dB | 最陡 |
| **汉宁窗** | 较窄 | -44dB | 中等 |
| **汉明窗** | 较窄 | **-53dB** | 中等 |
| 布莱克曼窗 | 较宽 | -74dB | 较缓 |
| 凯塞窗 | 可调 | 可调 | 可调 |

**选 Hamming 的原因**：矩形窗阻带衰减太低（-21dB），滤波效果差；布莱克曼窗衰减大但过渡带太宽，需要更高的阶数。Hamming 在阻带衰减（-53dB）和过渡带宽度之间取得了最好的折中，是工程中最常用的窗函数之一。

代码中的系数正是用 MATLAB `fir1(100, 0.2, hamming(101))` 生成的。`0.2` 是归一化截止频率（= 1000Hz / 5000Hz，其中 5000Hz 是 Nyquist 频率）。

### Q3：截止频率为什么选 1kHz？阶数 101 怎么确定的？

原始设计要求：保留 500Hz 信号，滤除 3000Hz 信号。

- 通带边缘：~800Hz（500Hz 要无损通过）
- 阻带边缘：~1000Hz（3000Hz 要在阻带内）
- 采样率 10kHz → Nyquist 频率 5kHz
- 归一化截止频率 = 1000/5000 = **0.2**

阶数 101 是在 MATLAB 里试出来的：
- 阶数太低（~30）：过渡带太宽，3000Hz 衰减不够
- 阶数太高（>150）：STM32F103C8T6 的 20KB SRAM 放不下状态缓冲区
- 101 阶（100 + 1）：过渡带 ~200Hz，3000Hz 处衰减 ~53dB，状态缓冲区占 ~2.4KB

## 二、算法实现（追问重点）

### Q4：为什么不用 CMSIS-DSP 库，手动实现 FIR/FFT？

尝试过集成 CMSIS-DSP，但 Keil MDK ARMCC V5 下 `arm_fir_f32` 报链接错误：

- CMSIS-DSP 的 `arm_fir_f32` 函数在 Cortex-M3 上没有对应的内核优化实现（只对 M0/M7 做了特殊优化），`ARM_MATH_CM3` 宏定义会导致函数体被跳过
- 手动添加了 10+ 个源文件仍有依赖缺失
- **最终决定纯 C 手动实现**——反而更能展示对算法原理的理解

手动实现更让老师认可，因为：
1. 你真正理解了卷积运算 `y[n] = Σh[k]·x[n-k]`
2. 理解了 FFT 的蝶形运算和位反转
3. 理解了 Goertzel 算法的递推公式

### Q5：FIR 卷积的代码实现？讲一下状态缓冲区机制

```c
void fir_filter(float *in, float *out, int n) {
    // 1. 状态缓冲区左移（把旧数据移出去）
    for (int i = 0; i < FIR_NUM_TAPS - 1; i++)
        fir_state[i] = fir_state[i + n];

    // 2. 写入新样本到末尾
    for (int i = 0; i < n; i++)
        fir_state[FIR_NUM_TAPS - 1 + i] = in[i];

    // 3. FIR 卷积: y[n] = Σ h[k] · x[n-k]
    for (int i = 0; i < n; i++) {
        float sum = 0;
        for (int j = 0; j < FIR_NUM_TAPS; j++)
            sum += fir_coeffs[j] * fir_state[FIR_NUM_TAPS - 1 + i - j];
        out[i] = sum;
    }
}
```

**状态缓冲区**：FIR 滤波器的输出不仅依赖当前输入，还依赖前 `FIR_NUM_TAPS-1` 个历史输入。`fir_state` 就是一个长度为 `FIR_NUM_TAPS + FFT_SIZE - 1` 的环形缓冲区，每次处理完一帧数据后，把最后 `FIR_NUM_TAPS-1` 个样本保留到下一帧，保证帧与帧之间的连续性，避免帧边界出现跳变噪声。

### Q6：Goertzel 算法的原理？为什么用它而不是 FFT？

**Goertzel 算法**是一个专门计算**单个特定频率**幅值的算法，本质是一个二阶 IIR 滤波器，谐振在目标频率上。

```
递推公式:
  s0 = x[n] + 2·cos(2π·f/Fs)·s1 - s2
  s2 = s1, s1 = s0

处理完 N 个点后:
  幅值 = √(s1² + s2² - 2·cos(2π·f/Fs)·s1·s2)
```

**为什么用 Goertzel 而不是 FFT 算这 2 个频点？**
- FFT 算全部 512 个频点，但只需要 500Hz 和 3000Hz 这两个
- Goertzel 算一个频点的计算量 ≈ N 次乘加 = 512 次
- FFT 算全部频点的计算量 ≈ N·log₂N = 512×9 = 4608 次
- 算 2 个频点：Goertzel 是 1024 次，FFT 是 4608 次——**Goertzel 快 4.5 倍**

所以 Goertzel 适用于"只需要少数几个频点"的场景，比如 DTMF 信号解码。

### Q7：FFT 的手动实现原理？

使用 **Cooley-Tukey 基-2 时间抽取（DIT）FFT**：

1. **位反转**：将输入序列的索引按二进制位倒序重排（例如索引 3=011 → 110=6）
2. **蝶形运算**：从长度为 2 的小 DFT 开始，逐级合并为更长的 DFT
3. **旋转因子**：每级的旋转因子 `W_N^k = e^(-j·2πk/N)`

```
输入 x[0..N-1]
  ↓ 位反转
  ↓ 第 1 级：len=2，蝶形合并 2 点 DFT
  ↓ 第 2 级：len=4，蝶形合并 4 点 DFT
  ↓ ...
  ↓ 第 9 级：len=512，得到完整 FFT
  ↓ 幅值计算
输出 X[0..N/2-1] (单边谱)
```

总蝶形数：N/2 · log₂N = 256 × 9 = 2304 次复数蝶形运算，在 72MHz 下约 1ms 完成。

## 三、硬件设计（可能问）

### Q8：ADC 采样率 10kHz 怎么配置的？为什么用 TIM3 触发？

CubeMX 配置：
- 时钟树：HSE 8MHz → PLL×9 → SYSCLK 72MHz
- APB1 36MHz（TIM3 挂 APB1，实际时钟 72MHz 因为 APB1 预分频≠1 时 ×2）
- TIM3：PSC=71，ARR=99 → 72MHz / (71+1) / (99+1) = **10kHz**
- ADC 预分频：/6 → ADC 时钟 12MHz（STM32F103 要求 ≤14MHz）

TIM3 产生 Update Event → 触发 ADC1 启动转换 → 转换完成后 DMA 自动搬运结果 → 循环往复。

**为什么用 TIM3 而不用 TIM2？**
STM32F103C8T6 的 ADC1 不支持 TIM2 TRGO 触发（硬件限制），CubeMX 里 External Trigger 选项找不到 TIM2。

### Q9：DMA 的工作原理？为什么用 Circular 模式？

DMA 配置：
- DMA1 通道 1
- 外设→内存（ADC DR → `adc_dma[]`）
- Half Word（16bit，匹配 ADC 12bit 输出）
- **Circular**（循环）模式

**Circular 模式**：DMA 传输完 512 次后，地址自动回到数组开头重新开始，持续不断。配合 TIM3 连续触发 ADC，实现完全的**后台采集**——CPU 不需要干预采样过程，只管处理已采集好的数据。

### Q10：OLED 为什么用软件 I2C？为什么不用硬件 I2C？

STM32F103C8T6 的硬件 I2C 外设（I2C1：PB6/PB7）在 Bugu（F1 系列 I2C 硬件 bug）：
- 在特定时序下会误产生 STOP 信号
- SR 寄存器状态可能异常
- 需要复杂的错误恢复逻辑

**软件 I2C**（bit-bang）直接把 GPIO 当 I2C 用，时序完全可控，不存在硬件 bug，移植到任何平台都能用。代价是 CPU 占用高——刷一屏 1024 字节约 100ms。

### Q11：信号调理电路怎么设计的？

```
信号源 → [1μF 隔直] → 1.65V 偏置 (10k+10k 分压) → PA0 (ADC输入)
```

ADC 输入范围 0~3.3V。外部信号可能是交流信号（正负摆动），直接接入会超出 ADC 范围：
1. **1μF 隔直电容**：滤除信号的直流分量
2. **两个 10kΩ 分压**：从 3.3V 得到 1.65V 直流偏置
3. 交流信号叠加在 1.65V 偏置上，ADC 输入始终在 0~3.3V 内

## 四、项目实现细节（深问）

### Q12：为什么采样 512 点？做 1024 点不好吗？

SRAM 只有 20KB，1024 点的缓冲区占用：
- `adc_dma[1024]` = 2KB
- `buf_a[1024]` + `buf_b[1024]` + `fir_state[...]` = ~12KB
- 合计 ~18.4KB → **超出 SRAM 容量**

512 点：
- `adc_dma[512]` = 1KB
- `buf_a[512]` + `buf_b[512]` + `fir_state[612]` = ~6.4KB
- 合计 ~9.4KB → **留有余量**

频率分辨率：10000 / 512 = **19.53Hz**，分辨 500Hz 和 3000Hz 绰绰有余。

### Q13：Goertzel 输出幅值为什么用 V_SCALE 归一化？

Goertzel 算法输出的原始值不是电压，而是 `N/2` 倍的信号幅值（N=512 时，单位幅值正弦输出 256）。归一化系数：

```
V_SCALE = 2000 / FFT_SIZE = 3.90625
```

把原始 Goertzel 输出 ×V_SCALE，相当于 ÷256 再 ×1000，结果可以直接用整数显示：
- 幅值 1.0V → 1.000V
- 幅值 0.5V → 0.500V

### Q14：为什么串口不用 %f 打印浮点数？

Keil MDK 使用 MicroLIB 时，`%f` 格式化需要链接浮点打印支持，会显著增加代码体积。避免方法：把浮点数先乘 1000 取整，再分别输出整数部分和小数部分。

```c
int val = (int)(voltage * 1000);
sprintf(str, "%d.%03d", val / 1000, val % 1000);
// 输出: "1.000"
```

### Q15：OLED 显示文字为什么全是大写？

`oled.c` 的 6×8 字库只定义了 `0x20~0x5A`（空格、数字、大写字母、常见符号），不包含小写字母。用小写字母（如 `"Atten"`、`"Fs10k"`）会访问到字库外的内存，显示乱码。所以全部用大写：`"ATTEN"`、`"3K"`。

### Q16：为什么 OLED 要先画频谱再叠文字？反过来不行？

```c
show_spectrum(...);          // 清屏 → 画柱 → (不更新)
OLED_ShowString(...);        // 叠文字到 framebuffer
OLED_Update();               // 一次性刷到 OLED
```

`show_spectrum` 内部会调用 `OLED_Clear()` 清空帧缓冲。如果在调用 `show_spectrum` 之前就写文字，文字会被清掉。所以顺序是：先画频谱（清屏 + 绘图），再叠文字，最后一次性 `OLED_Update()`。

### Q17：为什么 VLA 会导致栈溢出？怎么发现的？

`fft_mag()` 函数中原来的代码：
```c
void fft_mag(float *in, float *out, int n) {
    float real[n], imag[n];  // n=512 → 各 2KB, 共 4KB 栈空间
```

而 `startup_stm32f103xb.s` 定义的栈大小：
```asm
Stack_Size EQU 0x400    ; 仅 1KB
```

4KB 的 VLA 远超 1KB 的栈，导致**栈溢出**——写到了 BSS 全局变量区，把 `fir_state`、`buf_a` 等缓冲区覆盖了，造成信号处理结果异常、OLED 显示数据全是垃圾。

修复：把 VLA 换成全局静态数组 `fft_real[512]` 和 `fft_imag[512]`。

## 五、测试与验证

### Q18：怎么验证滤波器正常工作？

输入测试信号 `x(t) = sin(2π·500t) + 0.5·sin(2π·3000t)`：

1. **看串口数据**：
   ```
   500:1.000V->1.000V 3K:0.500V->0.008V -35.6dB
   ```
   500Hz 幅值不变（通带），3000Hz 从 0.5V 降到 0.008V（衰减 -36dB）

2. **看 OLED 频谱**：500Hz 处有明显柱峰，3000Hz 处几乎看不到柱子

3. **对比测试**：
   - 使用 3500Hz 单频信号：全部被滤除，OLED 几乎没有柱子
   - 使用 300Hz 单频信号：几乎无衰减通过

4. **关掉滤波器对比**：如果把 FIR 系数改为全通（中间系数=1，其他=0），3000Hz 幅值不会下降

### Q19：阻带衰减实测多少？和理论值差多少？

- 理论设计值：~53dB（Hamming 窗理论阻带衰减）
- 软件仿真实测：~51~52dB
- 硬件实测：~48~52dB（取决于信号质量和噪声）

偏差主要来自 ADC 量化噪声（12bit，理论 SNR ≈ 74dB）和信号源精度。结果基本符合预期。

### Q20：软件信号模式和 ADC 模式怎么切换？

在 `main.c` 顶部修改宏定义：
```c
#define USE_ADC  0   // 0 = 软件信号（无需硬件，方便演示）
                     // 1 = ADC 采样（需连接信号源和硬件）
```

软件模式内部生成正弦波，ADC 模式通过 TIM3+DMA 采集外部信号，其余处理逻辑（FIR、Goertzel、FFT、OLED、串口）完全共用。

## 六、进阶追问（加分项）

### Q21：如果要把截止频率改成 2kHz，需要改哪些参数？

1. MATLAB 重新生成系数：`fir1(100, 0.4, hamming(101))`（归一化截止频率 = 2000/5000 = 0.4）
2. 替换 `fir_coeffs.h` 中的系数数组
3. OLED 显示不用改，串口格式不用改

### Q22：如果信号有直流偏置会影响滤波结果吗？

FIR 高通/低通滤波器对直流分量的响应取决于系数之和。对于低通滤波器，系数之和 = 1（直流增益 = 1），直流分量会通过。但对于 Goertzel 和 FFT，直流分量只影响 0Hz 频点（`out[0]`），不影响 500Hz 和 3000Hz 的测量。

代码中先将 ADC 原始值转换为电压（0~3.3V），包含了直流偏置，但不影响 AC 成分的幅值测量。

### Q23：这个滤波器能处理实时信号吗？延迟多少？

- FIR 处理一帧（512 点）：~2ms
- FFT：~1ms
- Goertzel（2 个频点）：<0.5ms
- 总处理延迟：<5ms

软件 I2C 的 OLED 刷新最耗时间（~100ms），但这个属于显示延迟，不影响信号处理本身。ADC 模式下，DMA 后台采集不受 OLED 影响，每 51.2ms 采集完一帧就处理。

### Q24：Bootloader 和这个项目的关系？

（如果老师看到你 GitHub 上还有 bootloader 项目可能会问）

那个 STM32  bootloader 是另一个独立项目，支持 USART/USB/SD 三种方式的固件升级，带 AES-128 加密和 CRC32 校验。和这个滤波器项目没有直接关系，只是都用 STM32F103。

## 七、可能会被问到的细节陷阱

| 可能的陷阱问题 | 回答要点 |
|--------------|---------|
| "ADC 是 12bit 的，为什么用 float 存？" | 计算过程中需要小数精度，最终显示时转回整数 |
| "1024 字节的 OLED 帧缓冲放哪了？" | 静态全局数组，在 BSS 段，不占栈空间 |
| "为什么测量 500Hz 和 3000Hz 而不是其他频率？" | 测试信号只包含这两个频率，验证通带和阻带 |
| "采样 512 点够不够？" | 500/19.53≈25.6 个 bin，3000/19.53≈153.6 个 bin，分辨率足够 |
| "PA8 引脚的 LED 有什么用？" | 调试用，程序运行时会以帧率闪烁，直观指示运行状态 |
| "为什么不用实时操作系统？" | 单任务轮询即可满足需求，RTOS 增加复杂度 |
| "OLED 不接会怎样？" | 初始化检测 I2C 无应答，设置 `oled_ok=0`，所有 OLED 函数静默跳过，不影响信号处理 |
| "软件 I2C 速度多少？" | ~50kHz，由 `i2c_delay()` 的空循环次数控制 |
